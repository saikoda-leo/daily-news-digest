# Milestone v1.0: Full-Text Article Breakdown

**Status:** ✅ SHIPPED 2026-05-12
**Phases:** 1–2
**Total Plans:** 6

## Overview

Two phases deliver the full-text structured breakdown feature. Phase 1 wires in article fetching and structured Claude summarization — the backend pipeline that produces `core_idea` and `key_points` per RSS article. Phase 2 updates the HTML renderer to display that structured data in the article dropdowns, completing the user-visible feature.

## Milestone Summary

**Key Decisions:**
- Replace `ai_summary` string with `core_idea` + `key_points` fields (clean data model, no dual-shape items)
- Use trafilatura instead of BeautifulSoup for extraction (higher quality; lxml_html_clean required as transitive dep)
- Assistant prefill `"["` forces JSON array output from Claude, eliminating markdown-fence stripping
- Parallel fetch via ThreadPoolExecutor(10) before summarization; all article I/O completes first
- Structured prompt batched per section (one call per RSS section, not one per article)
- `_safe_structured()` normalizer pads/truncates `key_points` to exactly 5, wraps string values

**Issues Resolved:**
- Double-unescape needed in `_strip_html()` — entity-encoded HTML from mock RSS required pre-unescaping before HTMLParser
- Character-counting tests replaced with substring presence check (`"x"*6000 in prompt`) to avoid template character collisions
- `lxml_html_clean` missing as transitive dep for trafilatura 2.0.0 — required separate `pip install`

**Issues Deferred:**
- None

**Technical Debt Incurred:**
- `lxml_html_clean` not pinned in `requirements.txt` (transitive dep; pip resolves on install)
