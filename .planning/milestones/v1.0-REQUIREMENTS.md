## v1 Requirements (All Complete)

### Article Fetching

- [x] **FETCH-01**: System fetches full HTML of each RSS article URL using `requests` with a 10-second timeout
- [x] **FETCH-02**: System extracts clean article body text from HTML using `trafilatura`
- [x] **FETCH-03**: System falls back to RSS feed summary text if fetch fails, times out, or extracted text is fewer than 200 characters
- [x] **FETCH-04**: Extracted article text is truncated to 6000 characters before being sent to Claude
- [x] **FETCH-05**: All RSS article fetches run concurrently via `ThreadPoolExecutor` before summarization begins

### Structured Summarization

- [x] **SUMM-01**: Claude produces a structured JSON object per article: `{"core_idea": "...", "key_points": ["...", "...", "...", "...", "..."]}`
- [x] **SUMM-02**: Claude calls are batched per section (one call per RSS section, not one call per article) to minimize API usage
- [x] **SUMM-03**: If Claude returns malformed JSON or a `key_points` list with the wrong length, system falls back to empty structured summary (no crash)
- [x] **SUMM-04**: Existing 1-2 sentence `ai_summary` string field on RSS items is replaced by `core_idea` (str) and `key_points` (list[str]) fields

### HTML Rendering

- [x] **REND-01**: Core idea sentence is always-visible below each article title, outside the `<details>` dropdown, so the user can scan all stories without expanding anything
- [x] **REND-02**: Expanding an article dropdown shows a numbered list (1–5) of key points
- [x] **REND-03**: "Read full article" link appears at the bottom of every expanded dropdown (unchanged behavior)
- [x] **REND-04**: If an article has no structured summary (fallback case), no core idea line is shown and the dropdown degrades gracefully to a plain link (no broken layout)

## Out of Scope

| Feature | Reason |
|---------|--------|
| Apply structured breakdown to GitHub/Reddit items | Those link to repos and threads, not articles; the format doesn't fit |
| Paywall bypass (cookie injection, archiver services) | Legal and technical complexity out of proportion to benefit |
| Headless browser for JS-rendered pages | Adds 150-300 MB binary deps to GitHub Actions for marginal gain |
| Separate Claude call per article | Too expensive; batch-per-section approach keeps costs proportional |
