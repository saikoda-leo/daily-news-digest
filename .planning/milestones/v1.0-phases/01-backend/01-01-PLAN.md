1. Create `src/scrapers/article.py` with EXACTLY this content:
```python
import sys
from concurrent.futures import ThreadPoolExecutor

import requests
import trafilatura

_HEADERS = {"User-Agent": "Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36"}

SKIP_URL_PATTERNS = (".pdf", "youtube.com/watch", "youtu.be/", "github.com/")
MIN_CONTENT_CHARS = 200


def _fetch_one(item: dict) -> None:
    """Mutates item in place: sets item['full_text'] (str, may be empty). Never raises."""
    url = item.get("url", "")
    if not url or any(p in url for p in SKIP_URL_PATTERNS):
        item["full_text"] = ""
        return
    try:
        resp = requests.get(url, timeout=(3.05, 8), headers=_HEADERS)
        resp.raise_for_status()
        if resp.encoding and resp.encoding.lower() in ("iso-8859-1", "latin-1"):
            resp.encoding = resp.apparent_encoding or "utf-8"
        if "text/html" not in resp.headers.get("Content-Type", ""):
            item["full_text"] = ""
            return
        text = trafilatura.extract(
            resp.text,
            include_comments=False,
            include_tables=False,
            favor_recall=True,
        )
        item["full_text"] = text if (text and len(text) >= MIN_CONTENT_CHARS) else ""
    except Exception as e:
        print(f"[warn] article fetch {url}: {e}", file=sys.stderr)
        item["full_text"] = ""


def fetch_article_texts(items: list[dict]) -> None:
    """Mutates all items in place concurrently via ThreadPoolExecutor(max_workers=10). Never raises."""
    with ThreadPoolExecutor(max_workers=10) as executor:
        list(executor.map(_fetch_one, items))
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| External news sites → fetcher | Untrusted HTML returned by article URLs from RSS feeds (which the user configures in feeds.yaml) |
| trafilatura → in-memory item dict | Extracted article text becomes part of an item dict that is later passed verbatim into a Claude prompt (Plan 02) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Tampering (SSRF) | _fetch_one URL handling | accept | URLs originate exclusively from feeds the user configures in feeds.yaml; no user-supplied input from outside the local environment. Personal-tool low-risk per RESEARCH security domain. |
| T-01-02 | DoS | _fetch_one binary download | mitigate | Skip URLs matching SKIP_URL_PATTERNS (.pdf, youtube.com/watch, youtu.be/, github.com/) before HTTP call; reject responses where Content-Type does not contain "text/html"; requests.get uses timeout=(3.05, 8) |
| T-01-03 | DoS | fetch_article_texts concurrency | mitigate | ThreadPoolExecutor capped at max_workers=10 (not number-of-items); prevents connection exhaustion and rate-limit triggers |
| T-01-04 | Information Disclosure | requests.get error messages | accept | Exception messages include the URL in stderr warnings — acceptable for a personal local tool whose stderr is owner-only |
| T-01-05 | Tampering (prompt injection prep) | trafilatura output → item dict | accept | Extracted article text may contain prompt-injection attempts; downstream Claude output is HTML-rendered for the owner only and triggers no privileged actions. Documented in RESEARCH security domain. |
</threat_model>
